# PostgreSQL Migration Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Replace SQLite auto-migration with PostgreSQL and explicit SQL migrations while preserving simple local frontend/backend/database development.

**Architecture:** The backend will connect through `DATABASE_URL` and stop owning schema rollout. PostgreSQL schema state will live in versioned SQL files under `db/migrations`, local infrastructure will move into `infra/docker/compose.local.yml`, and helper scripts will bootstrap local infra, run migrations, and optionally seed development data.

**Tech Stack:** Go, Gin, GORM models, PostgreSQL, `golang-migrate`, Docker Compose, Vite/React, shell scripts
